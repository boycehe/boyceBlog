Xcode使用技巧

Xcode使用技巧

本文介绍常用的Xcode使用技巧，Xcode主要基于8.3.2版本，但这些技巧基本是比较稳定的，历经多个版本。

###快捷键
首先我们要认识下Mac上的键盘和文章符号对应关系：

下边我们介绍我们比较常用的快捷键，我的原则是快捷键只有用了才能记住，太多的快捷键我们也很难记住，所以我只推荐较少的快捷键。

  • Command+Shift+O 搜索文件并打开
  • Command+1~8： 分别对应上边从左右到右的菜单
  • Command+0： 展示/隐藏导航区域
  • Command+option+J： 光标跳到过滤导航输入框
  • Command+shift+Y： 显示/隐藏调试区
  • Command+option+0： 显示/隐藏实用工具区
  • Command+option+1~n： 切换实用工具区选项卡
  • Command+shift+k： 清除缓存文件
  • Command+Shift+Option+K： 清除所有生成的product缓存文件
  • Control+6：方法的跳转菜单
  • Control+5：当前文件下的文件列表
  • Control+command+Up 和 Control+command+Down 头文件和实现文件切换
  • Control+Command+J： 查看符号来源

###代码片段
添加代码片段，需要在你所添加的代码段中有<##>这样符号块，##中间可以添加文字
然后选中代码片段拖到代码片段框中，步骤如下图：

  • 编写相关代码
  • 将代码拖入代码片段框
  • 输入关键字

在设置代码块时有几个属性如下图所示：

Title:代码片段名称
Summary：代码片段简介
Platform:选定限定平台，只有在相关平台热键才会有效
Languge：选定限定语言，只有在相关语言热键才会有效
Completion Shortcut:热键编辑
Completion Scopes:热键作用的范围

###Xcode 断点设置
Xcode断点是每个iOS开发者必须掌握的调试方法，可以用快捷键Command＋\在该行添加断点;断点设置如下图所示：

Condition:
这里可以设置断点的条件，里边只能识别断点作用域内的变量，但不能识别预处理的宏定义；
Ignore:
指的忽略的次数；
Action:
指的是断点触发后执行的动作包括：AppleScript，Capture GPU Frame，Debugger Command，Log Message，Shell Command，Sound；AppleScript：是苹果开发的一种脚本
Capture GPU Frame：
触发断点后捕获GPU当前所绘制的帧。该功能是辅助图形调试的
Debugger Command：
触发断点后执行相应的lldb命令
Log Message：
触发断点后再debug框打印相应的信息
Shell Command：
触发断点后执行shell
Sound：
触发断点后播放声音

###异常断点
这个是很常用的断点类型，对于大多数的闪退都可以较为准确的定位到有问题的代码，用起来也非常简单，只需要才选中即可，如下图所示：

  • 选中导航栏中的断点项
  • 然后点击+号，选择Exception Breakpoint

    运行工程，如果遇到闪退，则会停在在出现闪退代码边。异常断点应该属于最常用的类型断点了，用起来也是简单有效的，如果有些闪退并没有跳转到相应的代码片段，可以尝试下边的方法，勾选

###LLDB调试
从Xcode4.0后调试器就换成了lldb，lldb是一款非常强大的调试器，掌握lldb调试命令可以让你调试的事半功倍，lldb用起来非常方便，只需要设置断点，执行到断点所在位置时就可以在debug框中输入相应的lldb命令，而lldb常用的命令有以下这些：

  • po:打印对象或者方法返回的值
  • p:打印变量的值

  • e:修改变量的值

  • image：主要用于寻址,较为常用的是寻找地址的代码位置如下图寻找闪退代码的行数：
